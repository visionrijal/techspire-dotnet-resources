⊞ Windows

.NET 10 & VS Code
Installation Guide

Follow each step in order. All commands run in a single PowerShell window.

1

Open PowerShell as Administrator

Press Win + S, type PowerShell, right-click the result and choose Run as Administrator.

🖼️ Screenshot goes here powershell.png
⚠️
Administrator requiredRunning as Administrator lets winget install software system-wide and allows changing the execution policy.
2

Set PowerShell Execution Policy Required

Windows blocks scripts by default. Paste this command and press Enter:

powershell 
Set-ExecutionPolicy Unrestricted -Scope CurrentUser -Force
ℹ️
What this does-Scope CurrentUser limits this change to your account only. Other users on the machine are unaffected.
3

Confirm if Prompted

If you see a confirmation prompt, type Y and press Enter to proceed.

🖼️ Screenshot goes here execution-policy.png
4

Install the .NET 10 SDK winget

winget is Windows' built-in package manager. The SDK includes the runtime, ASP.NET Core, and the dotnet CLI — no separate downloads needed.

powershell 
winget install Microsoft.DotNet.SDK.10
🖼️ Screenshot goes here winget.png
✅
This installs everythingThe .NET SDK bundles the base runtime and ASP.NET Core runtime. You do not need to install the runtime separately.
5

Install VS Code & C# Extensions

Run these commands one by one. After VS Code installs, open a new terminal window so the code command becomes available, then run the extension command.

powershell — VS Code 
winget install Microsoft.VisualStudioCode
powershell — C# Dev Kit extension 
code --install-extension ms-dotnettools.csdevkit
🖼️ Screenshot goes here image.png
ℹ️
C# Dev Kit installs two extensionsIt automatically pulls in the base C# (Roslyn) extension as a dependency — you get IntelliSense, error checking, and a Solution Explorer out of the box.
✓

Done! Verify your installation

Open a new PowerShell window (no need for Administrator this time) and run:

powershell 
dotnet --version
dotnet --list-runtimes

You should see output like this:

10.0.x
Microsoft.AspNetCore.App 10.0.x [C:\Program Files\dotnet\shared\...]
Microsoft.NETCore.App 10.0.x [C:\Program Files\dotnet\shared\...]

Both AspNetCore.App and NETCore.App appearing confirms the SDK and runtimes installed correctly.
